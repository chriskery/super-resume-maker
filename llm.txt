# Super Resume Maker

> 基于 React + Express 的 Web 端简历生成器，支持多模板切换、实时预览、PDF 导出、AI 润色和本地 JSON 文件存储。

## 项目架构

- **前端**: Vite 5 + React 18 + TypeScript + Tailwind CSS
- **后端**: Express.js (Node.js)
- **存储**: JSON 文件 (`./data/resumes/`)
- **PDF导出**: react-to-print (浏览器打印引擎)
- **AI润色**: OpenAI 兼容接口 (环境变量配置)

## API 接口

### 简历 CRUD
- `GET /api/resumes` — 列出所有简历摘要
- `GET /api/resumes/:id` — 获取单份简历完整数据
- `POST /api/resumes` — 新建简历
- `PUT /api/resumes/:id` — 更新简历
- `DELETE /api/resumes/:id` — 删除简历

### AI 润色
- `POST /api/ai/polish` — 文本润色
  - Body: `{ text: string, type: 'summary' | 'highlight' | 'skill' }`
  - Response: `{ result: string }`

## 数据模型

```typescript
interface Resume {
  id: string;
  title: string;
  templateId: string;            // 'professional' | 'minimal' | 'modern'
  personalInfo: PersonalInfo;    // name, phone, email, title, photo?
  summary: string;
  workExperience: WorkExperience[];
  projectExperience: ProjectExperience[];
  organizationExperience: OrganizationExperience[];
  awards: string[];
  skills: Skill[];               // { category, items }
  createdAt: string;
  updatedAt: string;
}
```

## 快速启动

```bash
npm install && npm run install:all
npm run dev
# 前端: http://localhost:5174
# 后端: http://localhost:3001
```

## 部署命令

```bash
make sync      # rsync 同步到远程服务器
make start     # 远程启动应用
make test      # 运行前后端测试
```

## 模板系统

模板通过 `registry.ts` 注册，新增模板只需：
1. 创建模板组件 (接收 `{ resume: Resume }` prop)
2. 在 registry 数组中添加一项

## 关键技术决策

- PDF 导出使用 react-to-print 而非 @react-pdf/renderer，避免维护两套模板代码
- 存储使用本地 JSON 文件而非 localStorage，满足"运行目录存储"需求
- Vite proxy 代理 API 请求，开发时前后端分离无跨域问题
- 模板组件使用 inline style 确保打印时样式不丢失
